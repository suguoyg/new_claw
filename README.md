# Multi-Agent Architecture Project

## 项目概述

本项目旨在构建一个**多Agent协作架构**的对话助手系统，类似 OpenClaw。系统采用前后端分离设计，打包为**单一服务**部署。

## 技术栈

| 层级 | 技术选型 |
|------|----------|
| **后端** | Python |
| **前端** | Vue.js |
| **通信** | REST API + 流式 WebSocket |
| **Agent通信** | 消息队列（轻量级：Redis Pub/Sub） |
| **持久化** | Markdown 文件存储 |
| **记忆检索** | 向量数据库（本地：Chroma） |
| **部署** | Docker + 纯服务器 |
| **用户系统** | 简单（用户名/密码） |

```

│            Tool/Skill 异步并行 + 流式返回                     │
└─────────────────────────────────────────────────────────────┘

  用户请求
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│ 1. Agent 接收请求，分析意图                                  │
│ - 需要调用哪些 Tool / Skill？                               │
│ - 制定执行计划                                              │
└─────────────────────────────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│ 2. WebSocket 流式发送执行状态                               │
│ 发送: {"type": "executing", "name": "web_search"}          │
└─────────────────────────────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│ 3. 并行调度 Tool/Skill（不等待结果）                        │
│                                                              │
│ ┌──────────────┐  ┌──────────────┐  ┌──────────────┐      │
│ │ Tool A      │  │ Skill B     │  │ Tool C      │      │
│ │ 执行中...    │  │ 执行中...    │  │ 执行中...    │      │
│ │ 流式推送状态 │  │ 流式推送状态 │  │ 流式推送状态 │      │
│ └──────────────┘  └──────────────┘  └──────────────┘      │
└─────────────────────────────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────────────────────────────┐
│ 4. 实时流式返回执行状态到前端                               │
│                                                              │
│ {"type": "tool_start", "name": "web_search"}               │
│ {"type": "tool_status", "name": "web_search", "status": "running", "progress": "50%"}│
```

## Agent 配置文件

| 文件 | 内容 | 说明 |
|------|------|------|
| **AGENTS.md** | Agent基本配置 | 名称、描述、关联模型、状态等 |
| **SOUL.md** | Agent灵魂定义 | 角色定位、人格特征、行为约束 |
| **USER.md** | 用户配置 | 当前对话用户的信息 |
| **MEMORY.md** | 记忆配置 | 用户自定义的记录内容/规则 |
| **TOOLS.md** | 工具配置 | 工具说明（方法名、描述） |
| **SKILL.md** | Skill配置 | Skill信息 + 执行规则 |
| **HEARTBEAT.md** | 心跳配置 | 保持为空 |

## 启动方式

### 方式1: Docker (推荐)
```bash
./scripts/build_docker.sh
# 访问 http://localhost:8000
```

### 方式2: 本地开发（分离模式）
```bash
# 终端1 - 后端
cd backend && uvicorn main:app --reload

# 终端2 - 前端
cd frontend && npm run dev
```

### 方式3: 本地开发（前后端共用端口）
```bash
# 1. 构建前端
cd frontend && npm run build

# 2. 启动后端（自动服务前端静态文件）
cd backend && python main.py
# 访问 http://localhost:8000
```
